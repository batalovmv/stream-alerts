# Task Records

Use this directory tree for durable task history.

## Files

- `backlog.md` - accepted work that is not started
- `active/` - work in progress
- `done/` - finished work
- `templates/task.md` - task template

## Naming

Use `YYYY-MM-DD-slug.md`.

Examples:

- `2026-03-08-ai-collaboration-foundation.md`
- `2026-03-10-max-provider-delivery-retry.md`

## Minimum Fields

Each non-trivial task record should capture:

- goal
- scope
- affected files or areas
- constraints or assumptions
- validation performed
- result or remaining blocker
